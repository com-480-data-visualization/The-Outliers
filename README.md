## Regenerating the prototype data

The two JSON files under `prototype/data/` (`satellites.json` and
`satellites-globe.json`) are derived from the UCS Satellite Database CSV in
`data/`. They are committed to the repo so the website can be served
statically, but they should never be edited by hand — re-run the build
script instead:

```bash
pip install -r scripts/requirements.txt
python scripts/build_data.py            # rebuild both JSONs
python scripts/build_data.py --check    # verify committed JSONs are up to date
python scripts/test_build_data.py       # run the test suite
```

On top of the aggregation, the script cleans up a few UCS data issues:
orbits where the label contradicts the altitude (LEO satellites sitting at
36,000 km), eccentricities outside `[0, 1)` that are clearly sign-error
typos (`5.11E+02` should be `5.11E-02`), and a handful of rows with
corrupt parameters that get dropped entirely. Everything it touches is
logged to [`data/data_quality_flags.md`](data/data_quality_flags.md).

`--check` recomputes the JSONs in memory and compares against the
committed files, exiting non-zero on any drift.
